# /incident - 인시던트 대응 파이프라인

장애 발생 시 신속한 대응을 위한 자동화된 흐름입니다.

## 사용법
```
/incident [상황 설명]
```

## 예시
```
/incident 서비스 전체 다운
/incident 에러율 급증 (10% 이상)
/incident 응답시간 5초 이상
```

## 파이프라인 흐름

```
1. respond-incident
   └→ 상황 파악 & 즉각 대응
   └→ 필요시 rollback

2. rollback (필요시)
   └→ 이전 버전으로 복구

3. monitor
   └→ 안정화 확인

4. diagnose
   └→ 근본 원인 분석

5. postmortem
   └→ 사후 분석 문서 작성
```

## 심각도별 대응

| 심각도 | 증상 | 대응 시간 |
|--------|------|----------|
| SEV1 | 전체 다운 | 5분 이내 |
| SEV2 | 주요 기능 장애 | 15분 이내 |
| SEV3 | 부분 기능 저하 | 1시간 이내 |

## 핵심 원칙
```
1. 복구가 먼저, 분석은 나중
2. 확실하지 않으면 롤백
3. 비난 없는 문화 (Blameless)
```

## 인시던트 종료 후
- 반드시 postmortem 작성
- Action Items 추적
- 재발 방지 대책 수립
