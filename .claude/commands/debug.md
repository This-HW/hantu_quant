# 디버깅 실행

**즉시 실행하세요. 설명하지 말고 바로 실행합니다.**

에러 정보: $ARGUMENTS

---

## 1단계: 에러 분석 (즉시 실행)

에러 메시지에서 파악:
- 에러 유형 (문법/타입/런타임/빌드)
- 파일 위치 및 라인 번호
- 스택 트레이스

해당 파일을 Read로 확인하고 컨텍스트 파악

## 2단계: 원인별 처리

### 단순 에러 (타입/문법/오타)
→ Edit tool로 바로 수정

### 외부 의존성 문제
→ WebSearch로 최신 정보 확인:
```
query: [패키지명] [에러메시지] solution 2024
```
→ 해결책 적용

### 복잡한 로직 문제
→ 관련 코드 탐색 (Grep/Glob)
→ 수정 방안 도출 후 적용

## 3단계: 수정 적용

```
- 최소 변경 원칙
- exc_info=True 포함 (Python)
- 에러 핸들링 추가 (필요시)
```

## 4단계: 검증

수정 후 관련 테스트 또는 빌드 실행:
```bash
# 테스트
pytest [관련파일] -v

# 또는 빌드
npm run build / go build / etc.
```
