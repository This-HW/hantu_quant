## 디버깅 프로세스

### 1단계: 에러 분석
`debug-issues` subagent를 사용하여:
- 에러 메시지 분석
- 스택 트레이스 파악
- 루트 원인 식별

### 2단계: 원인 분류

에러 유형에 따라 분기:

#### 단순 에러 (타입/문법/오타)
→ 바로 수정 진행

#### 외부 의존성 문제
→ WebSearch로 최신 정보 확인 후 수정

#### 복잡한 로직 문제
→ 관련 코드 탐색 후 수정 방안 제시

### 3단계: 수정
- 최소 변경 원칙
- 관련 테스트 확인

### 4단계: 검증
`verify-code` subagent로 수정 확인
