# /project-sync - 프로젝트 간 동기화

여러 프로젝트의 공통 요소를 동기화합니다.

---

## 사용법

```bash
/project-sync                    # 전체 프로젝트 동기화 상태 확인
/project-sync patterns           # 패턴 동기화
/project-sync deps               # 의존성 동기화
/project-sync configs            # 설정 동기화
/project-sync [project] [target] # 특정 프로젝트로 동기화
```

---

## 동기화 대상

### 1. 패턴 동기화

```
공유 가능한 패턴:
- 에이전트 정의 (.md)
- 스킬 정의 (skill.md)
- 훅 스크립트 (.py)
- 규칙 파일 (.md)
```

### 2. 의존성 동기화

```
동기화 항목:
- package.json 버전 통일
- requirements.txt 버전 통일
- 보안 패치 일괄 적용
```

### 3. 설정 동기화

```
동기화 항목:
- .claude/settings.json
- 린터/포맷터 설정
- CI/CD 설정
```

## 출력 예시

```markdown
# 🔄 프로젝트 동기화 리포트

## 분석 결과

| 프로젝트    | 패턴   | 의존성 | 설정   |
| ----------- | ------ | ------ | ------ |
| hantu_quant | 최신   | 구버전 | 최신   |
| project_b   | 구버전 | 최신   | 구버전 |
| project_c   | 최신   | 최신   | 최신   |

## 동기화 필요 항목

### 의존성

- hantu_quant: lodash 4.17.20 → 4.17.21 (보안 패치)

### 패턴

- project_b: review-code.md 업데이트 필요

### 설정

- project_b: .claude/settings.json 동기화 필요

## 제안 액션

1. `lodash` 보안 패치 일괄 적용
2. `review-code.md` 최신 버전 배포
3. 설정 파일 동기화
```
