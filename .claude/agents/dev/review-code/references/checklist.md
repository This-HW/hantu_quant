# 리뷰 체크리스트

## 설계 & 구조

- [ ] 단일 책임 원칙 준수
- [ ] 적절한 추상화 수준
- [ ] 의존성 방향 적절
- [ ] 중복 코드 최소화

---

## 코드 품질

- [ ] 변수/함수명 명확
- [ ] 함수 길이 적절 (20줄 이하 권장)
- [ ] 중첩 깊이 적절 (3단계 이하)
- [ ] 매직 넘버/스트링 없음

---

## 에러 처리

- [ ] 예외 상황 처리됨
- [ ] 에러 메시지 명확
- [ ] 실패 시 상태 일관성 유지
- [ ] 로깅 적절

---

## 보안

- [ ] 사용자 입력 검증
- [ ] SQL/XSS 인젝션 방지
- [ ] 민감 정보 노출 없음
- [ ] 권한 검사 적절

---

## 성능

- [ ] 불필요한 연산 없음
- [ ] N+1 쿼리 없음
- [ ] 메모리 누수 가능성 없음
- [ ] 적절한 캐싱

---

## 테스트

- [ ] 핵심 로직 테스트 있음
- [ ] 엣지 케이스 테스트 있음
- [ ] 테스트가 실제로 검증함

---

## 출력 템플릿

```markdown
### 리뷰 요약

#### 전체 평가: ✅ Approve / 🔄 Request Changes / ❓ Questions

**변경 개요:**
- 목적: [변경 목적]
- 범위: [영향 범위]
- 복잡도: [Low/Medium/High]

### 피드백 상세

#### 🔴 Must Fix (반드시 수정)
**[MF-1]** 파일: `path/to/file.ts:42`
**문제**: [문제 설명]
**해결**: [해결 방안]

#### 🟡 Should Fix (수정 권장)
**[SF-1]** 파일: `path/to/file.ts:78`
**문제**: [문제 설명]
**제안**: [개선 방안]

#### 🟢 Consider (고려 사항)
**[C-1]** [고려 사항]

#### 👍 Praise (좋은 점)
- [좋은 점]

### 통계
| 항목 | 개수 |
|------|------|
| Must Fix | N개 |
| Should Fix | N개 |
```
