# MCP 서버 활용 가이드

> MCP(Model Context Protocol) 서버를 상황에 맞게 활용하여 최신 정보와 전문 도구를 사용합니다.

---

## MCP 서버 목록

### 1. Context7 (문서/라이브러리)

**용도:** 최신 라이브러리 문서 및 코드 예제 조회

```
사용 시점:
- 라이브러리 사용법이 필요할 때
- 최신 버전 API가 궁금할 때
- 공식 문서 기반 구현이 필요할 때
```

**사용 방법:**
```
1. resolve-library-id로 라이브러리 ID 확인
2. query-docs로 필요한 정보 조회
```

**예시:**
```
Q: "Next.js 14에서 서버 컴포넌트 사용법"
→ context7로 Next.js 공식 문서에서 최신 정보 조회
```

---

### 2. Magic MCP - 21st.dev (UI/UX)

**용도:** 최적화된 UI 컴포넌트 및 디자인 패턴

```
사용 시점:
- 프론트엔드 UI 컴포넌트 구현 시
- 디자인 시스템 구축 시
- UX 패턴 참고가 필요할 때
- 애니메이션/인터랙션 구현 시
```

**활용 예시:**
```
- 버튼, 모달, 폼 등 기본 컴포넌트
- 다크모드, 테마 시스템
- 반응형 레이아웃 패턴
- 마이크로 인터랙션
```

---

### 3. 기타 MCP 서버 (확장 가능)

| MCP 서버 | 용도 | 사용 시점 |
|----------|------|----------|
| GitHub MCP | 이슈, PR 관리 | 협업 워크플로우 |
| Figma MCP | 디자인 연동 | 디자인 → 코드 변환 |

## MCP 활용 원칙

### 1. 최신 정보 우선

```
❌ 기억에 의존한 구현
   "내가 알기로 React 18에서는..."

✅ MCP로 확인 후 구현
   "context7로 React 18 공식 문서 확인 후..."
```

### 2. 상황별 MCP 선택

```
┌─────────────────────────────────────────┐
│  작업 유형          │  사용할 MCP        │
├─────────────────────────────────────────┤
│  라이브러리 사용법   │  Context7          │
│  UI 컴포넌트 구현    │  Magic MCP         │
│  API 스펙 확인      │  Context7          │
│  디자인 참조        │  Magic MCP / Figma │
│  기획서 확인        │  Notion MCP        │
└─────────────────────────────────────────┘
```

### 3. 버전 명시

```
MCP 조회 시 버전을 명시하세요:

✅ "Next.js 14.x App Router 사용법"
✅ "React 18 Suspense 패턴"
✅ "TypeScript 5.x satisfies 연산자"

❌ "Next.js 사용법" (버전 불명확)
```

## 프론트엔드 개발 시 MCP 활용

### UI 컴포넌트 구현 워크플로우

```
1. 기획/디자인 확인
   └→ Figma MCP 또는 기획 문서 참조

2. 컴포넌트 패턴 조회
   └→ Magic MCP (21st.dev)로 최적 패턴 확인

3. 라이브러리 API 확인
   └→ Context7로 사용할 라이브러리 문서 조회

4. 구현
   └→ 조회한 정보 기반으로 구현
```

### 예시: 모달 컴포넌트 구현

```
Step 1: Magic MCP로 모달 UI 패턴 확인
        - 접근성 고려사항
        - 애니메이션 패턴
        - 키보드 네비게이션

Step 2: Context7로 사용 라이브러리 확인
        - Radix UI Dialog 사용법
        - Framer Motion 애니메이션

Step 3: 기획서와 대조
        - 요구사항 충족 확인
        - 디자인 스펙 확인

Step 4: 구현 및 테스트
```

## 백엔드 개발 시 MCP 활용

### API 개발 워크플로우

```
1. 프레임워크 문서 확인
   └→ Context7 (FastAPI, Express, NestJS 등)

2. ORM/DB 문서 확인
   └→ Context7 (Prisma, TypeORM, SQLAlchemy 등)

3. 인증/보안 패턴
   └→ Context7 (JWT, OAuth 등)
```
