# 한투 퀀트 프로젝트 로드맵

## 프로젝트 개발 방식 변경

### 기존 방식의 문제점
- 너무 큰 프로젝트를 한번에 시작하려고 함
- 복잡한 기능들이 얽혀있어 디버깅과 테스트가 어려움
- 실제 동작 검증이 어려움

### 새로운 단계별 접근 방식
점진적이고 검증 가능한 5단계 개발 방식으로 변경

## 현재 진행 상황

### 완료된 기능
- ✅ 프로젝트 기본 구조 설정
- ✅ API 연동 기본 프레임워크
- ✅ 모의투자 API 연동 안정화
- ✅ 토큰 관리 시스템 구축
- ✅ 단계별 프로젝트 구조 재설계
- ✅ 개발 가이드라인 수립

### 현재 작업 중
- 🔄 Phase 2: 일일 매매 주식 선정 준비

## 단계별 개발 계획

### Phase 1: 감시 리스트 구축 ✅ 완료 (2025-07-09)
**목표**: 좋은 기업을 찾아서 매매 희망 리스트 혹은 감시 리스트에 추가하기

#### 주요 작업
- ✅ 기업 스크리닝 로직 구현
  - ✅ 재무제표 기반 스크리닝 (ROE, PER, PBR, 부채비율, 매출성장률, 영업이익률)
  - ✅ 기술적 분석 기반 스크리닝 (이동평균, RSI, 거래량, 모멘텀, 변동성)
  - ✅ 모멘텀 기반 스크리닝 (상대강도, 다기간 모멘텀, 거래량 모멘텀, 섹터 모멘텀)
  
- ✅ 감시 리스트 관리 시스템
  - ✅ 감시 리스트 CRUD 기능
  - ✅ 종목 추가/삭제/수정 기능
  - ✅ 감시 이유 및 메모 관리
  - ✅ 통계 정보 및 리포트 기능
  
- ✅ 기업 평가 엔진
  - ✅ 종합 평가 점수 계산 (가중치 기반)
  - ✅ 섹터별 비교 분석
  - ✅ 평가 기준 동적 조정 (시장 상황 반영)

#### 완료 기준
- ✅ 기본 스크리닝 로직 구현 (StockScreener)
- ✅ 감시 리스트 CRUD 기능 (WatchlistManager)
- ✅ 종합 평가 시스템 (EvaluationEngine)
- ✅ CLI 워크플로우 구현 (Phase1Workflow)
- ✅ 단위 테스트 통과 (17개 테스트 모두 통과)

#### 구현된 기능
- **StockScreener**: 재무/기술/모멘텀 기반 종목 스크리닝
- **WatchlistManager**: 감시 리스트 관리 및 통계
- **EvaluationEngine**: 가중치 기반 종합 평가
- **Phase1Workflow**: CLI 기반 워크플로우 (screen, list, add, remove, report)

### Phase 2: 일일 매매 주식 선정 (예상 기간: 1-2주)

  - [ ] 거래량 분석 (거래량 급증, 평균 거래량 대비)
  
- [ ] 일일 업데이트 스케줄러

  - [ ] 감시 리스트 → 매매 리스트 필터링
  - [ ] 시장 상황 고려한 선정 로직
  
- [ ] 선정 기준 관리
  - [ ] 동적 기준 조정 시스템
  - [ ] 백테스트 기반 기준 최적화
  - [ ] 시장 변동성 고려한 기준 변경

#### 완료 기준

- 최대 손실 제한 설정

- 정기적인 성과 검토

## 성공 지표

### 기술적 성공 지표
- 시스템 가동률 99% 이상
- API 호출 성공률 95% 이상
- 실시간 처리 지연 시간 < 1초

### 투자 성과 지표
- 연간 수익률 > 시장 수익률
- 최대 낙폭(MDD) < 20%
- 샤프 비율 > 1.0

### 운영 효율성 지표
- 일일 모니터링 시간 < 30분
- 수동 개입 빈도 < 주 1회
- 시스템 오류율 < 1%
