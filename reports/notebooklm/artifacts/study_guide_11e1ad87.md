# 한투 퀀트 시스템 종합 학습 가이드

본 가이드는 한국투자증권 Open API를 활용한 AI 기반 주식 투자 자동화 플랫폼인 '한투 퀀트(Hantu Quant)' 시스템의 핵심 구조와 작동 원리를 파악하기 위해 제작되었습니다. 이 문서는 시스템 아키텍처, 단계별 알고리즘, 리스크 관리 및 성과 분석 체계를 포괄적으로 다룹니다.

---

## I. 시스템 핵심 개념 요약

한투 퀀트 시스템은 총 6개의 단계(Phase)를 거쳐 데이터 수집, 분석, 종목 선정, 매매, 피드백의 전 과정을 자동화합니다.

### 1. 단계별 운영 프로세스
*   **Phase 1 (스크리닝):** 전체 상장 종목(약 2,875개)을 대상으로 펀더멘털, 테크니컬, 모멘텀 지표를 분석하여 감시 리스트(Watchlist)를 생성합니다. 병렬 처리를 통해 5~6분 내 분석을 완료합니다.
*   **Phase 2 (일일 선정):** 감시 리스트 중 당일 매매할 최적의 종목(5~10개)을 선정합니다. 가격 매력도, 거래량 패턴, 섹터별 상한 등을 고려합니다.
*   **Phase 3 (매매 실행):** KIS API를 통해 실시간 주문을 집행합니다. 손절(-3~5%), 익절(+8~10%), ATR 기반 트레일링 스톱 등 정교한 매도 엔진(SellEngine)이 작동합니다.
*   **Phase 4 (AI 학습 및 최적화):** 17개의 고급 피처(기울기 9개, 볼륨 8개)를 추출하여 RandomForest, GradientBoosting 등 ML 모델을 학습시키고 파라미터를 최적화합니다.
*   **Phase 5 (실시간 모니터링):** 시장의 이상 징후를 8가지 유형으로 감지하고 7개 채널(텔레그램, 슬랙 등)로 알림을 전송합니다.
*   **Phase 6 (웹 인터페이스):** React와 FastAPI 기반의 대시보드를 통해 시스템 상태와 성과를 시각화하고 제어합니다.

### 2. 기술적 특징 및 아키텍처
*   **엔터프라이즈급 모듈화:** 의존성 주입(DI), 플러그인 아키텍처를 도입하여 모듈 분리도 98%를 달성했습니다.
*   **지속 학습(Continuous Learning):** 일일 성과 분석 결과가 모델에 자동 피드백되어 선정 정확도를 지속적으로 개선합니다.
*   **리스크 관리:** 켈리 공식(Kelly Criterion)을 통한 포지션 사이징, 서킷 브레이커(일일/주간/최대 낙폭)를 통해 자산을 보호합니다.
*   **백테스트 시스템:** 시계열 Train/Val/Test 분할과 데이터 누수 방지를 위한 'Purge Gap'을 적용하여 검증의 신뢰도를 높였습니다.

## II. 단답형 연습 문제 (Short-answer Quiz)

**Q1. Phase 1 스크리닝 단계에서 종목이 통과하기 위한 총점 기준과 지표별 완화 정책의 세부 내용은 무엇입니까?**
*   **정답:** 총점 60.0점 이상이며, 펀더멘털/테크니컬/모멘텀 세 지표 중 2개 이상이 45.0점 이상이어야 합니다.

**Q2. 리스크 관리에서 최적의 투자 분율을 결정하기 위해 사용하는 수학적 공식의 명칭은 무엇입니까?**
*   **정답:** 켈리 공식 (Kelly Criterion)

**Q3. Phase 4 AI 학습 시스템에서 사용하는 피처 엔지니어링의 총 개수와 각 범주별 개수를 작성하십시오.**
*   **정답:** 총 17개 (기울기 피처 9개, 볼륨 피처 8개)

**Q4. 백테스트 시 데이터 누수(Data Leakage)를 방지하기 위해 Train과 Test 경계 사이에 설정하는 격리 기간의 명칭은 무엇입니까?**
*   **정답:** Purge gap

**Q5. 시스템 안정성 관리를 위해 사용되는 기법으로, 특정 컴포넌트 오류 시 연쇄 장애를 막고 대체 함수를 실행하는 기능을 무엇이라 합니까?**
*   **정답:** 회로 차단기 (Circuit Breaker)

## III. 심화 학습을 위한 에세이 주제 (Essay Questions)

**주제 1. 시스템의 전략적 불일치와 수익성 개선 방안 분석**
*   **질문:** 현재 시스템의 Phase 1 재무 필터(ROE, PER 등)는 보수적인 장기 투자에 적합한 반면, Phase 3의 매도 로직(타이트한 손절)은 단기 모멘텀 전략에 가깝습니다. 이러한 전략적 불일치가 수익성에 미치는 영향을 논하고, '수익성 개선 로드맵'에 제시된 필터 완화 및 가중치 조정의 필요성을 서술하십시오.

**주제 2. AI 학습 시스템의 자기 진화(Self-Evolution) 메커니즘**
*   **질문:** 한투 퀀트 시스템이 일일 성과 분석(Daily Performance Analysis) 결과를 어떻게 다시 학습 데이터(Labeling)로 활용하는지 설명하고, 이러한 지속 학습 플로우가 선정 정확도(목표 90%) 달성에 기여하는 기술적 경로를 분석하십시오.

**주제 3. 백테스트 신뢰성 확보를 위한 기술적 장치**
*   **질문:** 단순 랜덤 시뮬레이션의 한계를 극복하기 위해 도입된 'KIS API 실데이터 기반 백테스트'와 'Walk-Forward Analysis'의 개념을 설명하고, 거래 비용(수수료, 세금, 슬리페이지) 반영이 실전 매매 전환 결정에 왜 결정적인 요소인지 논하십시오.

## IV. 중요 용어 사전 (Glossary)

| 용어 | 정의 및 설명 |
| :--- | :--- |
| **KIS API** | 한국투자증권에서 제공하는 Open API로, 실시간 주가 조회 및 주문 실행에 사용됨. |
| **Kelly Criterion** | 과거 승률과 손익비를 바탕으로 파산 위험을 최소화하면서 수익을 극대화하는 투자 비중을 계산하는 공식. |
| **ATR (Average True Range)** | 시장의 변동성을 측정하는 지표로, 트레일링 스톱(추적 손절) 설정 시 주요 기준으로 활용됨. |
| **MDD (Max Drawdown)** | 특정 기간 동안 포트폴리오의 고점 대비 최대 하락폭을 나타내며, 시스템의 리스크를 측정하는 핵심 지표. |
| **Purge Gap** | 시계열 데이터 학습 시 미래 데이터가 과거 학습에 영향을 주지 않도록 Train 데이터와 Test 데이터 사이에 두는 시간적 격리 구간. |
| **DI Container** | 의존성 주입(Dependency Injection)을 관리하는 도구로, 모듈 간의 결합도를 낮추어 유지보수성과 확장성을 높임. |
| **Walk-Forward Analysis** | 데이터 윈도우를 이동시키며 학습과 검증을 반복하는 방식으로, 실제 시장 환경과 유사한 조건에서 전략의 전진 안정성을 검증함. |
| **Slippage** | 주문 시 희망했던 가격과 실제 체결된 가격 사이의 차이로, 거래 비용 계산 시 필수적으로 고려해야 함. |
| **pykrx** | 네이버/KRX 등에서 주가 데이터를 스크래핑하여 제공하는 라이브러리로, 시스템 내에서 실데이터 보완 및 검증용으로 사용됨. |
| **Sharpe Ratio** | 위험 대비 수익성을 나타내는 지표로, (수익률 - 무위험수익률) / 변동성 공식을 통해 계산됨. |

## 5. 핵심 용어 사전 (Glossary)

| 용어 | 정의 및 시스템 내 역할 |
| :--- | :--- |
| **KIS API** | 한국투자증권에서 제공하는 Open API로, 주문, 잔고 조회 및 실시간 시세 데이터 수집의 핵심 창구. |
| **Kelly Criterion** | 수익 확률과 손익비를 활용해 자산 대비 최적의 투자 금액 비율을 산출하는 공식. |
| **ATR (Average True Range)** | 시장의 변동성을 측정하는 지표. 매도 엔진에서 동적 손절선과 트레일링 스톱의 기준으로 사용됨. |
| **Walk-Forward Analysis** | 시계열 데이터를 Rolling Window 방식으로 나누어 학습과 검증을 반복, 과적합(Overfitting)을 방지하는 검증법. |
| **Purge Gap** | Train 데이터와 Test 데이터 사이의 시간 간격을 두어 미래 데이터가 학습에 포함되는 것을 방지하는 기술. |
| **DI (Dependency Injection)** | 객체 간의 의존 관계를 외부에서 주입하는 설계 패턴으로, 시스템의 결합도를 낮추는 데 기여함. |
| **Sharpe Ratio** | 위험(변동성) 대비 수익률을 나타내는 지표. 백테스트 결과의 질을 평가하는 주요 메트릭. |
| **Slippage** | 주문 시 희망가와 실제 체결가 사이의 차이. 백테스트 시 거래 비용으로 반드시 반영되어야 함. |
| **Anomaly Detector** | 실시간 시장 감시 중 급등락, 거래량 급증 등 8가지 유형의 이상 패턴을 탐지하는 모듈. |
| **pykrx** | 한국거래소(KRX) 데이터를 스크래핑하여 실제 주가 정보를 보완하는 데 사용하는 라이브러리. |
